You are browsing IPFS.

This entire page its HTML, this stylesheet, the logo, and the link you're about to click was fetched as content-addressed blocks and re-derived against its CID by a service worker before it rendered. No gateway was trusted. This is the dweb, and it loads like the web.

Content-addressed

The page's address is the hash of its bytes. Change a byte and the address changes so tampering is impossible to hide.

Trustless

The browser is the verifier. A wrong or malicious block simply fails re-derivation and is refused (Law L5).

Serverless

No origin server owns this site. It lives as a Merkle-DAG of blocks that any peer or your own node can serve.

One address space

The same CID is a did:holo and a holo:// name on the UOR substrate. IPFS isn't bridged here it's adopted.
