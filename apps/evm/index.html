Why this was impossible before · the UOR superpower

              
A normal fork (Hardhat/Anvil) trusts the RPC it believes whatever state the node returns. Holo EVM doesn't: it pulls an eth_getProof Merkle proof for every account & storage slot the EVM reads, and re-derives that they hash up to the state root committed in the block header a header it independently re-derives from raw consensus bytes (RLP→keccak). If the RPC lies by one byte, a proof fails to root and the call is flagged. The result: execute real mainnet contracts in your browser, trusting no server replay history, or run counterfactual “what-if” calls, all over cryptographically-verified state.

            

          

        

        
Reads any contract by forking over a public RPC, but verifies, never trusts: state is Merkle-proven against a re-derived block root (Law L5). Old blocks need an archive RPC; recent blocks work on any full node.

The witness is content-addressed: its κ = keccak256 of its canonical bytes, so a single flipped byte changes the κ and the proofs stop rooting. Verification re-derives the block header, re-roots every proof, re-checks keccak(code)==codeHash, and re-runs the EVM with the network physically disabled. Impossible with normal tooling: an explorer hands you data you must trust; a κ-witness hands you a proof your own device re-derives.

What this proves · The Verge, fully realized

              
A κ-witness (previous tab) verifies one call. This verifies an entire block's state transition: given the complete pre-state, the engine re-executes every tx and recomputes the Merkle-Patricia post-state root, which must equal the one committed in the block header the exact check a validating node performs. The vectors are the official ethereum/tests suite (the EF ground truth every client is measured against), vendored and κ-pinned. No node, no network: a reference-client conformance check, in a tab.

              
Live mainnet blocks need a producer-supplied witness (Glamsterdam block-level access lists, H2 2026) or an archive trace this verifier is ready for those the moment they ship.

            

          

        

        
Deterministic & offline: every vector is self-contained, so reproduction is pure execution + Merkle math (Law L5). The same engine that runs your contracts here passes the Ethereum consensus suite.
