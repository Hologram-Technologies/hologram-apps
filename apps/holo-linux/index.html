The problem with normal software

Almost everything you run is something you're asked to trust. A server sends it. A company can change it tomorrow. It can break, get pulled, or quietly become something else — and you can't inspect it. Usually it won't even work without a connection.

What's different here

It runs on your device, not a server. The processor is built in software, running inside this web page. Turn off your internet and keep typing — it keeps working.

It's checked before it runs. Every part — the Linux kernel, the whole filesystem — has a fingerprint: a short code derived from its exact bytes. Change one byte and the fingerprint changes completely. Your browser re-computes it and refuses to start if anything is off.

It can't be tampered with unnoticed. Because the identity is the bytes, an altered or corrupted system simply won't boot. or hit ⚠ tamper test and watch one flipped byte stop it cold.

Your work stays — with no cloud. Files you create live on a real disk inside your browser; they survive a reboot, with no server storing them. then press ⟳ reboot and run it again.

Anyone can reproduce it, exactly. The same fingerprint rebuilds the identical machine for anyone, anywhere, forever — no server needed to hand it out.
